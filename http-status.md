# HTTP 状态码解释及返回场景 🌐

[English](/en/http-status.md) | [Japanese](/ja/http-status.md) | [Chinese](/zh/http-status.md)

以下是 HTTP 协议中常见状态码的详细说明及其返回场景。

## 1xx：信息性状态码
1xx 状态码表示请求已被接收，需要继续处理。

- **100 Continue**
  - **含义**：服务器已收到请求头，客户端应继续发送请求体。
  - **场景**：客户端发送大文件时，先发送请求头，等待服务器确认继续发送。

- **101 Switching Protocols**
  - **含义**：服务器同意客户端的协议切换请求。
  - **场景**：WebSocket 协议从 HTTP 切换时使用。

---

## 2xx：成功状态码 ✅
2xx 状态码表示请求成功。

- **200 OK**
  - **含义**：请求成功，服务器返回正常数据。
  - **场景**：GET 请求返回数据，POST 请求返回操作结果。

- **201 Created**
  - **含义**：请求成功且资源已被创建。
  - **场景**：POST 创建新资源后返回。

- **202 Accepted**
  - **含义**：请求已接收，但尚未处理完成。
  - **场景**：异步操作，服务器需要更长时间处理。

- **204 No Content**
  - **含义**：请求成功，但服务器没有返回任何内容。
  - **场景**：删除资源成功，但无需返回内容。

---

## 3xx：重定向状态码 🔀
3xx 状态码表示资源位置发生变化，客户端需进一步操作。

- **301 Moved Permanently**
  - **含义**：资源永久移动到新位置。
  - **场景**：URL 改版，永久重定向到新地址。

- **302 Found**
  - **含义**：资源临时移动到新位置。
  - **场景**：页面暂时跳转，仍需使用原地址访问。

- **304 Not Modified**
  - **含义**：资源未修改，客户端可使用缓存。
  - **场景**：客户端提供缓存版本的资源标识（如 ETag），服务器确认资源未更新。

---

## 4xx：客户端错误状态码 ❌
4xx 状态码表示客户端请求有问题。

- **400 Bad Request**
  - **含义**：请求格式错误或参数不正确。
  - **场景**：缺少必需参数或 JSON 格式错误。

- **401 Unauthorized**
  - **含义**：未提供认证凭据或凭据无效。
  - **场景**：访问需要登录或 Token，但未提供或错误。

- **403 Forbidden**
  - **含义**：服务器拒绝执行请求。
  - **场景**：用户无权限访问资源。

- **404 Not Found**
  - **含义**：请求的资源不存在。
  - **场景**：访问错误的 URL 或资源已删除。

- **405 Method Not Allowed**
  - **含义**：请求方法不被允许。
  - **场景**：使用 POST 请求访问只支持 GET 的接口。

---

## 5xx：服务器错误状态码 ⚠️
5xx 状态码表示服务器处理请求时出错。

- **500 Internal Server Error**
  - **含义**：服务器内部错误。
  - **场景**：代码异常、服务宕机等。

- **502 Bad Gateway**
  - **含义**：网关或代理服务器收到无效响应。
  - **场景**：反向代理服务器无法连接后端服务。

- **503 Service Unavailable**
  - **含义**：服务器暂时不可用（过载或维护中）。
  - **场景**：系统流量过大或正在维护。

- **504 Gateway Timeout**
  - **含义**：网关或代理服务器等待后端服务超时。
  - **场景**：反向代理等待后端响应过久。

---

以上状态码帮助我们理解 HTTP 请求的执行情况及潜在问题，可以结合实际开发需求选择适当处理逻辑。
